**agents/base_agent.py — Major rewrite (preserve structure, replace internals):**

Keep: AgentState TypedDict, _truncate, _log_decision, build_agent_graph structure, run_agent entry point, observe_node, check_done_node, should_continue.

**Add to AgentState:** `available_skills: list[dict]` (skill metadata for prompt), `game_phase: str` (current phase: "setup"/"battle").

**Replace `_parse_llm_response`** with `_parse_skill_call(text: str) -> dict | None`:
- Layer 1: Try `json.loads(text.strip())` — look for `{"name": str, "parameters": dict}`
- Layer 2: Regex `r'\{[^{}]*"name"[^{}]*\}'` to extract JSON block from mixed text
- Layer 3: Regex `r'"name"\s*:\s*"(\w+)"' + r'"parameters"\s*:\s*(\{[^}]*\})'` for partial JSON
- Layer 4: If all fail, return None (agent will re-reason with a hint)
Do NOT fall back to raw command execution. If parsing fails, the agent loops back to reason with a nudge.

**Replace `_build_prompt`** — new version:
- System message includes: role description, available_skills as JSON list (from state["available_skills"]), output format instruction: `Respond with JSON: {"name": "skill_name", "parameters": {"key": "value"}, "reasoning": "why this skill"}`, game phase context (setup vs battle), turn count.
- For red agent: add adaptive behavior instruction — "If past turn 70% of max_turns and you haven't achieved exploitation, escalate aggression. Try multiple attack vectors."
- For blue agent during setup: "Focus on hardening: apply firewall rules, harden SSH, check for SUID binaries."
- For blue agent during battle: "Balance detection and response. Poll for intrusion signs, then act on findings."
- Keep rolling observation window and findings in user message.

**Replace `act_node`** with `tool_dispatcher_node`:
- Parse LLM output using `_parse_skill_call`
- If parse returns None: append observation "Failed to parse skill call. Please respond with ONLY JSON: {\"name\": \"skill_name\", \"parameters\": {...}, \"reasoning\": \"...\"}". Do NOT increment turn_number. Return to reason.
- If skill name not in SKILL_REGISTRY: append observation "Unknown skill '{name}'. Available: {skill_names}". Do NOT increment turn. Return to reason.
- If skill found: call `execute_skill(name, params, orchestrator_url)` from registry. Log decision. Increment turn. Add result to observations and findings.
- Deduplication: track skill calls (name + params hash) instead of raw commands.

**Update `build_agent_graph`**: rename "act" node to "tool_dispatcher". Add conditional edge from reason: if `_parse_skill_call` finds a call -> tool_dispatcher, else -> back to reason (with max 3 consecutive parse failures before forcing a safe default skill like "scan_processes" for blue or "port_scan" for red).

**POST /game/start** — Start a new game:
- Initialize GameContext with phase=SETUP, start_time=time.monotonic()
- Initialize ScoringEngine with db logger

- Return {"points_awarded": int, "total_scores": dict, "game_over": bool, "win_reason": str | None}

**GET /game/narrative** — Get final narrative:
