  <name>Task 1: Blue hardening and detection skills</name>
  <files>skills/blue/__init__.py, skills/blue/harden.py, skills/blue/detect.py, skills/registry.py</files>
  <action>
**skills/blue/harden.py:**
- `block_ip(ip: str, orchestrator_url: str, agent_role: str = "blue") -> dict`: POST `ufw deny from {ip} to any` to orchestrator. Return `{"success": bool, "blocked_ip": ip}`.
- `harden_ssh(orchestrator_url: str, agent_role: str = "blue") -> dict`: POST multi-command via `sed -i 's/^#*PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config && sed -i 's/^#*PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config && systemctl restart sshd` to orchestrator. Return `{"success": bool, "changes": ["PermitRootLogin no", "PasswordAuthentication no"]}`.
- `fix_suid(binary_path: str, orchestrator_url: str, agent_role: str = "blue") -> dict`: POST `chmod u-s {binary_path}` to orchestrator. Return `{"success": bool, "binary": binary_path}`.

**skills/blue/detect.py:**
- `scan_processes(orchestrator_url: str, agent_role: str = "blue") -> dict`: POST `ps aux --no-headers` to orchestrator. Parse stdout into list of `{"user": str, "pid": int, "command": str}`. Filter out known system processes (init, sshd owned by root, systemd). Return `{"processes": [dict], "suspicious": [dict]}` where suspicious = processes owned by non-root non-system users OR listening on unusual ports.
- `tail_auth_log(orchestrator_url: str, agent_role: str = "blue", lines: int = 50) -> dict`: POST `tail -n {lines} /var/log/auth.log 2>/dev/null || tail -n {lines} /var/log/secure 2>/dev/null` to orchestrator. Parse for failed login attempts (regex: `Failed password`), successful logins (`Accepted`), new users (`useradd`), sudo usage. Return `{"raw_lines": int, "failed_logins": int, "successful_logins": [dict], "new_users": [str], "sudo_events": [str]}`.
- `list_users(orchestrator_url: str, agent_role: str = "blue") -> dict`: POST `awk -F: '$3 >= 1000 && $3 < 65534 {print $1":"$3":"$7}' /etc/passwd` to orchestrator. Parse into list of `{"username": str, "uid": int, "shell": str}`. Return `{"users": [dict], "count": int}`.

IMPORTANT: Detection skills must NOT flag the blue agent's own activity. The blue agent connects from a known IP on the blue-net Docker network. Add a constant `BLUE_AGENT_IP = "blue-agent"` (Docker DNS name) and filter it out of suspicious results where applicable (e.g., in scan_processes, skip connections from blue-agent).

Register all 6 skills in skills/registry.py with role="blue".
  </action>
  <verify>
`python -c "from skills.registry import get_skills_for_role; blue = get_skills_for_role('blue'); print(len(blue)); print([s['name'] for s in blue])"` prints 6 skills.
  </verify>
  <done>6 blue hardening+detection skills registered: block_ip, harden_ssh, fix_suid, scan_processes, tail_auth_log, list_users.</done>
