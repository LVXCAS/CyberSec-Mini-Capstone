**game/state.py:**
- Enum `GamePhase`: SETUP, BATTLE, CONCLUSION
- Frozen dataclass `GameContext` with fields: phase (GamePhase), start_time (float, monotonic), setup_duration (int, default 300), battle_duration (int, default 1200), red_kill_chain (list[str], default_factory=list — tracks completed stages: "recon", "exploit", "privesc", "persist"), blue_lockout (bool, default False), service_down_since (dict[str, float], default_factory=dict — service_name -> monotonic timestamp when it went down).
- Function `check_win_condition(ctx: GameContext, scores: dict) -> tuple[bool, str]`: Returns (game_over: bool, reason: str). Checks: (1) time expiry — elapsed > setup_duration + battle_duration, (2) red full kill chain — all 4 stages in red_kill_chain, (3) blue lockout (flag set by scoring engine when all attack paths blocked), (4) critical service down > 120 seconds. Return reason strings like "Time expired", "Red completed full kill chain", "Blue achieved lockout", "Critical service {name} down for 2+ minutes".
- Function `advance_phase(ctx: GameContext) -> GameContext`: Returns new GameContext with phase advanced (SETUP->BATTLE->CONCLUSION). Uses immutable pattern (create new dataclass instance).

**game/scoring.py:**
- Dict `POINT_VALUES` matching the research scoring schema: recon_complete=5, service_exploited=15, privesc_achieved=20, persistence_installed=20, full_kill_chain=40, vuln_patched=10, attack_detected=10, attacker_blocked=15, service_kept_up=5, lockout_achieved=40, red_undetected_action=5, blue_detected_stealthily=5, pivot_on_failure=3, correct_inference=3, adaptive_escalation=5.
- Class `ScoringEngine` with:
  - `__init__(self, db_log_fn)` — takes a function to log score events (dependency injection for testability)
  - `award(self, agent_role: str, event_type: str, description: str, turn_number: int) -> int` — looks up points in POINT_VALUES, calls db_log_fn, returns points awarded
  - `check_stealth_bonus(self, red_action_turn: int, current_turn: int, blue_detected: bool) -> bool` — if red action not detected within 2 turns, award red_undetected_action. Returns whether bonus was awarded.
  - `check_kill_chain_progress(self, stage: str, current_chain: list[str]) -> tuple[list[str], bool]` — returns (updated_chain, full_chain_complete). Stages must be in order: recon, exploit, privesc, persist.
  - `get_totals(self) -> dict` — returns {"red": int, "blue": int} from accumulated awards

**game/snapshots.py:**
- Function `take_snapshot(orchestrator_url: str) -> dict`: POST a set of diagnostic commands to orchestrator /execute (as role "system" — add "system" as an allowed role in safety filter that bypasses role restrictions). Commands: `ps aux --no-headers`, `ss -tlnp`, `cat /etc/passwd | awk -F: '$3>=1000'`, `ufw status 2>/dev/null || iptables -L -n 2>/dev/null`, `crontab -l 2>/dev/null`. Combine results into a single dict: `{"timestamp": iso_str, "processes": str, "listening_ports": str, "users": str, "firewall": str, "cron": str}`. Store via db `log_snapshot(json.dumps(snapshot))`.
- Class `SnapshotManager`: wraps `take_snapshot` on a timer. `__init__(self, interval: int = 60, orchestrator_url: str)`. `start()` method spawns a daemon thread that calls take_snapshot every `interval` seconds. `stop()` method sets a threading.Event to stop. Snapshots go through the same DB writer to avoid SQLite lock contention (per research pitfall #5).

**game/narrative.py:**
- Function `generate_narrative(score_events: list[dict], final_scores: dict, win_reason: str, game_duration_seconds: float) -> str`: Produces a story-format string as specified in CONTEXT.md. Structure:
  1. Opening: "Game Duration: X minutes Y seconds. Winner: {red|blue|draw}. {win_reason}."
  2. Key moments: iterate score_events chronologically, highlight pivots (pivot_on_failure), kills (attacker_blocked), breaches (service_exploited, privesc_achieved, persistence_installed). Format each as a sentence: "Turn {N}: {agent} {description} (+{points} pts)".
  3. Kill chain progress: "Red kill chain: {completed stages} / 4 stages".
  4. Final scores: "Red: {N} points | Blue: {N} points".
  5. Stealth/detection summary: count of stealth bonuses vs detection bonuses.
  6. Reasoning highlights: list any pivot_on_failure or adaptive_escalation events.
- Return as a single formatted string with newlines, readable in terminal.
