**game/loop.py:**

Main function `run_game(orchestrator_url: str = "http://localhost:8000", setup_minutes: int = 5, battle_minutes: int = 20, max_turns_per_agent: int = 30) -> dict`:

Uses `requests` to call orchestrator endpoints (NOT asyncio — keep it simple, both agents run sequentially anyway because of KoboldCpp).

**Game flow (sequential, not async):**

1. **Initialize:** POST /game/start. Log "Game starting."

2. **Setup phase (blue only):**
   - Import and call `run_agent("blue", blue_prompt, max_turns=setup_turns)` where setup_turns = setup_minutes * 60 // 45 (estimate ~45 sec per turn). Cap at 8 turns.
   - After each turn (modify run_agent to yield or use a callback — OR simpler: just set max_turns low and let it run). The simplest approach: call run_agent with max_turns for setup, let it complete.
   - **CRITICAL: Score blue setup actions.** After blue setup completes, iterate over blue_state's executed actions (from the decision log or state tracking) and call `score_blue_action()` for each one. Alternatively, wrap each setup turn with scoring: use `run_single_turn` in a loop instead of `run_agent`, calling `score_blue_action(blue_state, orchestrator_url)` after each turn. This ensures blue's hardening actions (firewall, SSH hardening, SUID fixes) are recorded in score_events and count toward GAME-02 scoring.
   - After blue setup: POST /game/advance to transition to BATTLE.

3. **Battle phase (alternating turns):**
   - Do NOT run both agents in parallel (sequential KoboldCpp). Instead, alternate: red gets 1 turn, then blue gets 1 turn, repeat.
   - Implement as a loop:
     ```python
     for turn in range(max_battle_turns):
         # Check time
         status = requests.get(f"{orchestrator_url}/game/status").json()
         if status["win_condition"]["game_over"]:
             break
         elapsed = status["elapsed"]
         if elapsed > (setup_minutes + battle_minutes) * 60:
             break

         # Red turn
         red_result = run_single_turn("red", red_state, orchestrator_url)
         score_red_action(red_result, orchestrator_url)

         # Check win condition after red
         status = requests.get(f"{orchestrator_url}/game/status").json()
         if status["win_condition"]["game_over"]:
             break

         # Blue turn
         blue_result = run_single_turn("blue", blue_state, orchestrator_url)
         score_blue_action(blue_result, orchestrator_url)
     ```
   - `run_single_turn` is imported from `agents.base_agent` (built in 02-04). It runs one iteration of observe->reason->tool_dispatch and returns updated state.

4. **Conclusion:**
   - POST /game/advance to CONCLUSION
   - GET /game/status for final scores
   - GET /game/narrative for story
   - Return {"scores": dict, "narrative": str, "win_reason": str, "turns_played": int}

**Helper functions:**
- `score_red_action(state: AgentState, orchestrator_url: str, scoring_engine)`: Examine last_result and executed skill. If skill was port_scan and found open ports -> POST /game/score with event_type="recon_complete". If ssh_brute succeeded -> "service_exploited". If find_suid found exploitable -> "privesc_achieved". If add_backdoor_user/install_cron/add_ssh_key succeeded -> "persistence_installed". Track kill chain stages. **After scoring the red action, call `scoring_engine.check_stealth_bonus(red_turn_number, current_turn, blue_detected)` to award stealth points if blue didn't detect the action within 2 turns.** Track a `last_red_action_turn` variable in the game loop; after each blue turn, check if `score_blue_action` awarded "attack_detected" — if so, set `blue_detected=True` for that window.
- `score_blue_action(state: AgentState, orchestrator_url: str)`: If harden_ssh/block_ip/fix_suid succeeded -> "vuln_patched". If tail_auth_log found intrusion -> "attack_detected". If kill_process/remove_user on red's artifacts -> "attacker_blocked". If check_service shows active -> "service_kept_up". **This function is called both during setup phase and battle phase** — blue's hardening during setup is scored the same way.

**Important:** The game loop runs OUTSIDE Docker (or in a dedicated game-controller container). It calls the orchestrator via HTTP. For now, design it to run from the host machine via `python -m game.loop`.

**Note on agents/base_agent.py:** If `run_single_turn` was not already exported by 02-04, add it here. It should run one observe->reason->tool_dispatch cycle using the compiled graph with a 1-turn limit. This is a minor addition to the existing base_agent, not a rewrite.
