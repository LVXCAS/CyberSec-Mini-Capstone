  <action>
    Create `scripts/` directory if it does not exist.

    Create `scripts/test-network-isolation.sh` (chmod +x) that:

    1. Requires containers to be running (check with docker compose ps, exit 1 if not).

    2. Tests that red-agent CANNOT reach blue-agent:
       `docker compose exec red-agent ping -c 1 -W 2 blue-agent 2>&1`
       Expected: failure (non-zero exit). If ping succeeds, FAIL.

    3. Tests that blue-agent CANNOT reach red-agent:
       `docker compose exec blue-agent ping -c 1 -W 2 red-agent 2>&1`
       Expected: failure. If ping succeeds, FAIL.

    4. Tests that red-agent CAN reach orchestrator:
       `docker compose exec red-agent ping -c 1 -W 2 orchestrator 2>&1`
       Expected: success.

    5. Tests that blue-agent CAN reach orchestrator:
       `docker compose exec blue-agent ping -c 1 -W 2 orchestrator 2>&1`
       Expected: success.

    6. Print summary: PASS/FAIL for each test. Exit 0 if all pass, exit 1 if any fail.

    Use clear output with test names, green/red indicators (using printf with ANSI colors).
    Script should be POSIX-compatible bash (#!/usr/bin/env bash, set -euo pipefail).
  </action>
  <verify>
    `chmod +x scripts/test-network-isolation.sh && bash -n scripts/test-network-isolation.sh` — syntax check passes.
    If containers are running: `./scripts/test-network-isolation.sh` exits 0.
  </verify>
  <done>Script exists, is executable, syntax-valid, and correctly tests all 4 network paths (2 must fail, 2 must succeed).</done>
</task>
