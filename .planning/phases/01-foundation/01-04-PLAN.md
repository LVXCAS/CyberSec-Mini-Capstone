  <action>
    Create agents/requirements.txt: langgraph, langchain-core, requests, pydantic

    Create agents/base_agent.py with a LangGraph StateGraph implementing the autonomous loop:

    State schema (TypedDict):
    - agent_role: str (red/blue)
    - turn_number: int
    - max_turns: int (default 15)
    - observations: list[str] (rolling — last 5 observations)
    - executed_commands: set[str] (for dedup)
    - findings: list[dict] (long-term memory — key findings persisted as JSON)
    - messages: list (chat history for LLM context)
    - done: bool

    Nodes:
    1. observe_node: Query orchestrator GET /decisions/{role} for last action result. If first turn, run initial recon command (role-dependent). Build observation string from command output + system state. Append to observations (keep last 5 — rolling window).

    2. reason_node: Build LLM prompt from:
       - System prompt (role-specific, injected by subclass)
       - Rolling context: last 5 observations (short-term memory)
       - Key findings summary (long-term memory — JSON list)
       - Executed commands list (for dedup awareness)
       - Turn number and remaining turns
       Constraints: total prompt tokens < 2048 (use trim_messages or manual truncation — count chars, assume 4 chars/token, cap at 8192 chars).
       Send to KoboldCpp at http://host.docker.internal:5001/v1/chat/completions with max_tokens=512, temperature=0.7.
       Parse response as JSON tool-call: {"tool": "execute", "args": {"cmd": "..."}, "reasoning": "..."}
       If JSON parsing fails, try to extract command from freeform text using regex.
       Add 60-second timeout on inference call.

    3. act_node:
       - Check if command already in executed_commands — if so, modify slightly or skip (dedup)
       - POST to orchestrator /execute with {agent_role, command, reasoning, turn_number}
       - Add command to executed_commands
       - Store result in observations for next turn
       - If result contains notable finding, add to findings list (long-term memory)
       - Increment turn_number

    4. check_done_node: If turn_number >= max_turns or done flag set, end. Otherwise loop back to observe.

    Graph edges: observe -> reason -> act -> check_done -> (observe | END)

    Decision logging: At each act_node, write a JSONL line to /app/data/decisions.jsonl:
    {"timestamp": iso8601, "agent_role": str, "turn": int, "command": str, "reasoning": str, "result_exit_code": int, "result_stdout_preview": str(first 200 chars), "was_blocked": bool}

    Function: run_agent(role: str, system_prompt: str, max_turns: int = 15) -> dict
  </action>

    (Full integration test in Task 2)
  </verify>
