  <name>Task 1: Safety filter + SSH executor + SQLite database</name>
  <files>
    orchestrator/safety_filter.py
    orchestrator/ssh_executor.py
    orchestrator/db.py
    orchestrator/requirements.txt

  <action>
    Create orchestrator/requirements.txt: fastapi, uvicorn, paramiko, pydantic

    Create orchestrator/safety_filter.py:
    - Function: validate_command(command: str, agent_role: str) -> FilterResult
    - FilterResult is a Pydantic model: {allowed: bool, command: str, reason: str | None, sanitized_command: str | None}
    - Blocklist patterns (regex): rm -rf /, mkfs, dd if=, shutdown, reboot, halt, poweroff, :(){, fork bomb patterns, >(\/dev\/sd), chmod 777 /, wget.*\|.*sh, curl.*\|.*sh
    - Role-based restrictions: blue-agent cannot run nmap; red-agent cannot run auditd/iptables (they shouldn't have each other's tools, but enforce at filter level too)
    - Log blocked commands (visible for demo — include the blocked command and reason in return)
    - Allow everything else — this is a blocklist approach, not allowlist

    Create orchestrator/ssh_executor.py:
    - Function: execute_command(command: str, timeout: int = 30) -> CommandResult
    - CommandResult: {stdout: str, stderr: str, exit_code: int, timed_out: bool, execution_time_ms: int}
    - Use Paramiko SSHClient to connect to battleground (hostname: "battleground", user: "admin", password from env var BATTLEGROUND_PASSWORD)
    - Set command timeout (default 30s). If timeout, kill the channel and return timed_out=True.
    - Truncate stdout/stderr to 4096 chars max (prevent context explosion)
    - Handle connection errors gracefully — return error in stderr, exit_code -1

    Create orchestrator/db.py:
    - SQLite database at /app/data/game.db
    - Tables:
      - decision_log: id, timestamp, agent_role, turn_number, command, reasoning, result_stdout, result_stderr, exit_code, was_blocked, block_reason
      - safety_log: id, timestamp, agent_role, command, allowed, reason
      - findings: id, timestamp, agent_role, finding_type, description, severity, raw_data
    - Functions: init_db(), log_decision(), log_safety_check(), log_finding(), get_recent_decisions(agent_role, n)
    - Use WAL mode for concurrent reads
  </action>
  <verify>
    python3 -c "from orchestrator.safety_filter import validate_command; r = validate_command('rm -rf /', 'red'); assert not r.allowed; print('PASS: blocklist works')"
    python3 -c "from orchestrator.safety_filter import validate_command; r = validate_command('nmap -sV 10.0.0.5', 'red'); assert r.allowed; print('PASS: allowed command passes')"
    python3 -c "from orchestrator.db import init_db; init_db(); print('PASS: DB initialized')"
  </verify>
  <done>Safety filter rejects blocklisted commands. SSH executor connects to battleground. SQLite schema created with all tables.</done>

  <action>
    Create orchestrator/main.py (FastAPI app):

    Endpoints:
    - GET /health: returns {"status": "ok"}
    - POST /execute: accepts {agent_role: str, command: str, reasoning: str, turn_number: int}
      1. Run safety filter on command
      2. Log safety check to DB
      3. If blocked: log decision with was_blocked=True, return {allowed: false, reason: str, blocked_command: str}
      4. If allowed: execute via SSH, log decision with full result, return {allowed: true, result: CommandResult}
    - GET /decisions/{agent_role}: returns recent decisions for an agent (for context building)
    - POST /findings: accepts {agent_role: str, finding_type: str, description: str, severity: str, raw_data: str}, logs to findings table

    On startup: init_db(), test SSH connection to battleground (log warning if fails, don't crash).

    Run with: uvicorn orchestrator.main:app --host 0.0.0.0 --port 8000

    Integration test (inline or separate script):
    - Start orchestrator (docker compose up orchestrator)
    - POST /execute with allowed command (e.g., "whoami") — verify 200 + result contains "admin"
    - POST /execute with blocked command ("rm -rf /") — verify 200 + allowed=false + reason populated
    - GET /decisions/red — verify logged entries appear
  </action>
