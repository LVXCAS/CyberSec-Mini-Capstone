  <action>
    Create docker-compose.yml with 4 services:

    Networks:
    - red-net: internal: true (red-agent + orchestrator only)
    - blue-net: internal: true (blue-agent + orchestrator only)
    - orchestrator-net: normal bridge (orchestrator + battleground)

    Services:
    - orchestrator: Python 3.11-slim, connected to red-net + blue-net + orchestrator-net. Expose port 8000. extra_hosts: host.docker.internal:host-gateway (for KoboldCpp on host at port 5001). Depends on battleground.
    - red-agent: Python 3.11-slim, connected to red-net only. Has nmap, netcat pre-installed. Depends on orchestrator.
    - blue-agent: Python 3.11-slim, connected to blue-net only. Has auditd, net-tools pre-installed. Depends on orchestrator.
    - battleground: Ubuntu 22.04, connected to orchestrator-net only. Expose SSH port 22 internally.

    Each Dockerfile installs minimal Python deps. Agent Dockerfiles copy agent code. Orchestrator Dockerfile copies orchestrator code. All use WORKDIR /app.

    Add healthcheck to battleground: ssh localhost -o ConnectTimeout=2
    Add healthcheck to orchestrator: curl -f http://localhost:8000/health

    Create placeholder main.py files so containers can start (orchestrator: FastAPI with /health endpoint; agents: simple loop that sleeps).
  </action>
  <verify>
    Run `docker compose config` to validate YAML.
    Run `docker compose build` to confirm all images build.
    Run `docker compose up -d` and verify all 4 services reach "healthy" or "running" state within 60s.
  </verify>
  <done>All containers build and start. Network config valid.</done>
