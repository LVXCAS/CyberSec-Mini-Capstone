    - Verify file size after download (expect ~5GB for Q4_K_M)

    - Use koboldcpp_cu11 binary (CUDA 11 for K80 compute 3.7 — NOT CUDA 12)
    - Required flags: --model path/to/gemma-4.gguf --port 5001 --host 0.0.0.0 --contextsize 4096 --jinja --gpulayers 999 --usecublas
    - Add --threads $(nproc) for CPU fallback layers
    - Add timeout detection: if KoboldCpp doesn't respond within 120s, print error and exit
    - Print nvidia-smi output before launch for VRAM baseline
    - Make both scripts executable (chmod +x)

    IMPORTANT: Use koboldcpp_cu11, NOT koboldcpp. The K80 is compute capability 3.7 which requires CUDA 11.
    IMPORTANT: --jinja flag is required for Gemma 4 chat template.
    IMPORTANT: KoboldCpp v1.111.2+ minimum version.
  </action>
  <verify>
    Run `bash inference/download_model.sh` — model file exists and is ~5GB.
    Run `nvidia-smi` — K80 visible with available VRAM.
    Run `bash inference/start_koboldcpp.sh` in background — server responds on port 5001 within 120s.
    `curl http://localhost:5001/api/v1/models` returns model info.
  </verify>
  <done>KoboldCpp running on K80 with Gemma 4 loaded, API accessible on port 5001.</done>
